# Phase 双层合同模板

每个 phase 需要两份文档：**定位合同** `plan/phases/phase-X.md` 和 **执行合同** `plan/execution/phase-X.md`。职责不可合并。

---

## 1. 定位合同：`plan/phases/phase-X-<slug>.md`

回答"这个阶段**是什么**"。

```markdown
# Phase X: <标题>

## 阶段定位

<这个阶段在整条链路里解决什么问题，一句话>

## 必带上下文

- plan/common.md
- Phase X-1 已完成

## 阶段目标

- <可验证的目标 1>
- <可验证的目标 2>
- <可验证的目标 3>

## 实施范围

- <允许涉及的模块/文件/能力 1>
- <允许涉及的模块/文件/能力 2>

## 本阶段产出

- <具体交付物 1>
- <具体交付物 2>

## 明确不做

- <最容易被误伤的越界项 1>
- <最容易被误伤的越界项 2>

## 完成判定

- <机器可检或逐项可勾的客观检查 1>
- <机器可检或逐项可勾的客观检查 2>
- <机器可检或逐项可勾的客观检查 3>

## 依赖关系

- 依赖 Phase X-1
```

### 写作铁律

- **"阶段目标"必须可验证**。写不出验证方式的目标，是无效目标，重写
- **"明确不做"通常比"阶段目标"更重要**——它保护边界，优先写
- **"完成判定"禁止出现主观词**：不要"良好"、"合理"、"基本完成"、"性能可接受"
- 完成判定里的每一条都应该能回答"我怎么证明它达到了"

---

## 2. 执行合同：`plan/execution/phase-X-<slug>.md`

回答"这次实施**能碰什么、不能碰什么**"。

```markdown
# Phase X 执行包

本文件不能单独使用。执行 Phase X 时，必须同时携带完整的 `plan/common.md` 和 `plan/phases/phase-X-<slug>.md`。

## 必带上下文

- `plan/common.md`
- `plan/phases/phase-X-<slug>.md`

## 执行目标

- <本次实施要解决的具体问题 1>
- <本次实施要解决的具体问题 2>

## 本次允许改动

- <路径/模块 1，精确到目录或文件级>
- <路径/模块 2>
- <路径/模块 3>

## 本次不要做

- <禁止涉及的文件/模块/重构 1>
- <禁止涉及的能力/feature 2>
- <禁止使用的 API/库 3>

## 交付检查

- <客观可勾选项 1，与 phases/ 的完成判定一致或更细>
- <客观可勾选项 2>
- <客观可勾选项 3>

## 执行裁决规则

- 如果 <某触发条件>，直接判定无效并回退
- 如果引入超出 <本 phase 范围> 的范围漂移，直接回退到规划边界
- 如果依赖私有 API / 第三方库 / 超范围模块，直接判定无效
```

### 写作铁律

- **execution 是围栏，不是脚手架**。不要写"第一步 A、第二步 B"——那是替 AI 思考，AI 会机械照做并越界
- **"允许改动"必须是路径级白名单**，不是"只要和本阶段相关就行"
- **"裁决规则"是一票否决**：触发后无论工作量多大都要回退，不接受"已经做了很多所以迁就一下"
- **交付检查比完成判定更细**：phases/ 的完成判定是目标视角，execution 的交付检查是实施视角

---

## 3. 对一份优秀 phase 对的自检

写完后逐项过一遍：

- [ ] 定位合同里的"阶段目标"每一条都有对应的验证方式
- [ ] "明确不做"里列出了最容易被 AI 误伤的越界项
- [ ] "完成判定"里没有主观词
- [ ] 执行合同的"允许改动"是路径级白名单
- [ ] "本次不要做"里明确排除了其他 phase 的范围
- [ ] "交付检查"每一条都能勾或不勾，没有灰色地带
- [ ] "执行裁决规则"里至少包含一条"范围漂移立即回退"
- [ ] 两份文档的"必带上下文"都包含 `plan/common.md` 和对方
- [ ] 整体工作量估计在 30–90 分钟的 AI 实施窗口内

任一项不过，重写。

---

## 4. 常见反模式

| 反模式 | 表现 | 修正 |
|--------|------|------|
| 完成判定写"功能跑起来了" | phase 覆盖多个验收点 | 拆成多个 phase |
| 允许改动写"涉及 XXX 功能的代码" | 不是白名单，AI 会发散 | 改为具体路径 |
| execution 写 10 步详细过程 | AI 会照做但越界 | 改为围栏式 |
| 两个 phase 反复改同一批文件 | 切分维度错 | 换维度重切 |
| phase 文档 < 200 字 | 颗粒太细 | 合并 |
| common.md 里写"第一阶段要做 X" | 混入了策略 | 下放到对应 phase |
